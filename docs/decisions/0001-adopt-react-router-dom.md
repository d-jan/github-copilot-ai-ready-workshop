# ADR-0001: クライアントサイドルーティングに react-router-dom を採用

- ステータス: 採用
- 日付: 2026-07-17

## 背景

商品カードから商品詳細ページ（`/product/:id`）への画面遷移と、
一覧への戻り導線が必要になった。SPA 内でのページ遷移を実現する仕組みが必要。

## 決定

`react-router-dom` v7 をランタイム依存として追加する。

## 理由

- React エコシステムで最も広く使われるルーティングライブラリである。
- Node ≥ 20、React ≥ 18 と互換性がある（当プロジェクトの要件を満たす）。
- `BrowserRouter`、`Routes`、`Link` など宣言的な API で既存構造を壊さずに導入できる。

## 却下した代替案

- **自前の状態管理による画面切り替え**: ルーティング相当の実装が必要で、URL 同期やブラウザ履歴の管理が煩雑。
- **Tanstack Router**: 高機能だが型安全ルーティングの学習コストが高く、ハンズオン教材として過剰。

## 影響

- バンドルサイズ: gzip 約 18KB 増加（react-router-dom 本体）。
- `main.tsx` に `BrowserRouter` ラッパーが追加される。
